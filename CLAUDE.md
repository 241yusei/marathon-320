## 📥 データを受け取ったときの標準手順（最重要）

ユーザーから **ランニング/体調データ（日次・週次どちらでも）を入力・追加された** とき、または既存データを更新したときは、
チャットでのコーチング・フィードバックに**加えて**、毎回つぎを実行する。

### 1. コーチング・フィードバック（チャット）
従来どおり、データを分析して所見・改善点・次アクションを日本語で返す。
判断基準は `docs/sub3-research.md` と下記ゲートに従う（睡眠最優先、ACWR 0.8〜1.3、HRV低下日は強度を下げる 等）。

### 2. サイトデータ更新（`js/data.js`）→ commit & push
- 日次ログ → `dailyLog` に1行追記（必要なら `recentRuns` にも）。
- 朝の HRV/RHR/睡眠/体重 → `dailyLog` ＋ `metrics`・`highlights` の最新値を更新。
- 週次集計・月別トレンド → `trends.series.*.data`、`metrics` を更新。
- 優先事項・判定が変われば `week.priorities` / `week.judgment` を書き換え。
- ゲート/フェーズ進捗があれば `gates[].progress` / `phases[].active` / `meta.phase` を更新。
- **必ず `meta.lastUpdated` を更新**。
- 変更は designated ブランチ（`claude/running-workout-analysis-zzgxcj`）に commit & push。

### 3. 練習メニューの再考（7日分）— 必要に応じて
データを見てメニュー調整が要ると判断したら、**今後7日分**のスケジュールを再設計する。
- `js/data.js` の `schedule`（曜日別メニュー）を更新し、チャットでも7日分を提示。
- 原則: 80/20（イージー中心）、ACWR 0.8〜1.3 で漸増、ラン日は隔日配置、睡眠/HRVゲート未通過なら強度ゼロ寄り。
- フェーズ（Phase 0〜3＋テーパー）と現在地に整合させる。

### 4. Notion へ追加
書き込み先は **「🏃 ランニングログ」データベース**（ユーザー指定：全データをここに集約）。
- データソース（collection）ID: `d74128cb-b5b5-48db-912e-e592c89a3344`
- 親ページ「藤井勇成大改造計画」: `37b8eb8a-b7fe-8120-b118-d68d2bb23a91`
- 日次の走行/練習 → DBに1行追加。プロパティ:
  - `タイトル`(title) … 例「6/16 イージー 4.54km」
  - `実施日`(date) … `date:実施日:start` にISO日付
  - `種類`(select) … イージー / ロング / テンポ / インターバル / MP走 / 筋トレ / ウォーク のいずれか
  - `距離km`(number)、`ペース(/km)`(text)、`平均HR`(number)、`RPE(1-10)`(number)
  - `メモ`(text) … その日の振り返り・所見の要点
- 7日メニューや週次サマリーも同DBに残す（行のページ本文に記載、または週次まとめ行として追加）。

### 5. 【日次データのとき限定】振り返りを Notion＋Googleカレンダーへ
日のデータを受け取ったときは、その日の **振り返り（フィードバック）内容** を次の両方に記入する。
- **Notion**: 上記ランニングログDBの当該行の `メモ`（要点）＋必要に応じページ本文に詳細。
- **Googleカレンダー「RUN！！！」**:
  - calendarId: `21f063de48729542a7529811473147bc9fba01df613b06b6c68b58d92ecf3a3a@group.calendar.google.com`
  - タイムゾーン: Asia/Tokyo
  - 実施日に終日（または実施時間帯）のイベントを作成し、タイトルに練習概要、説明欄にフィードバック内容を記入。

## ゲート（判断の前提）
- 睡眠が最優先。睡眠 7h×14日連続 ＋ HRV 7日平均 50ms 以上で減量開始ゲートが開く（それまで減量しない）。
- HRV < 45ms または睡眠 < 5.5h の日は即休養／強度を落とす。
- Phase 1 移行は「週4回×2週連続・週30km到達・🔴判定が週1回以下」。
