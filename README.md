# マラソン3:20切りプロジェクト — Webダッシュボード

`marathon_plan/` の情報を、Apple（iPhone 17 Pro ページ）の設計言語で 1 枚に集約した
ヘルスケア・ダッシュボードサイト。**今週の目標・計画／ランニング結果・フィードバック／
目標達成に必要な情報**を蓄積し、データが来るたびに更新していく。

## 開き方

ビルド不要の静的サイト。`site/index.html` をブラウザで開くだけ。
ローカルサーバーで見る場合:

```bash
cd "藤井勇成（ヘルスケア）"
python3 -m http.server 8931 --directory site
# → http://localhost:8931
```

（Claude Code のプレビューからは `marathon-site`（`.claude/launch.json`）で起動）

## 構成

```
site/
├── index.html        ← セクション骨格（中身は JS が描画）
├── css/style.css     ← Apple 風スタイル（ダーク/ライト交互・大見出し・スクロール連動）
├── js/data.js        ← ★単一データソース（ここだけ編集すれば全体が更新される）
└── js/main.js        ← data.js を読み込み各セクションを描画・チャート・アニメ
```

## ★ 更新のしかた（蓄積とアップデート）

サイトの中身はすべて **`site/js/data.js`** から自動描画される。
DASHBOARD.md と同じ「最新状態の単一情報源」の Web 版。新しいデータが来たら
`data.js` の該当箇所を編集して保存 → ブラウザを再読み込みするだけで反映される。

| 何が来たか | data.js のどこを編集するか |
|------------|---------------------------|
| 朝のHRV/RHR/睡眠/体重 | `dailyLog` に1行追記 ＋ `metrics`・`highlights` の最新値を更新 |
| ランニング実績 | `dailyLog`（または `recentRuns`）に追記 |
| 週次の集計・月別トレンド | `trends.series.*.data` に追記、`metrics` を更新 |
| 今週の優先事項が変わった | `week.priorities` / `week.judgment` を書き換え |
| 体重・睡眠ゲートの進捗 | `gates[].progress` を更新（減量ゲートは 0〜14日） |
| フェーズ移行 | `phases[].active` を移動、`meta.phase` を更新 |
| 最新研究 | `research` に1件追加 |
| 何か更新したら | **必ず `meta.lastUpdated` を更新**（ナビ右上に表示される） |

> Claude（コーチ）にデータを送れば、COACHING.md のルールに従って
> DASHBOARD.md・logs/ と一緒に **この data.js も同期更新**される。

## セクション一覧

1. ヒーロー（目標タイム・減量・レースまでのカウントダウン）
2. ハイライト（最重要6指標のカード）
3. 現在地 / Status（最新値グリッド）
4. 今週の目標・計画（判定・優先事項・週間スケジュール）
5. ランニング結果・フィードバック（日次ログ・直近の走り）
6. 6ヶ月トレンド（HRV / RHR / 距離の切り替えチャート）
7. ゲート / 開放条件（減量・Phase移行）
8. 計画ロードマップ（Phase 0〜3＋テーパー）
9. 3:20切りに必要なもの（必要指標・タイム短縮の積み上げ）
10. 栄養・心拍ゾーン
11. 最新研究の反映
12. 強みとリスク・履歴ハイライト
13. 次の目標（CTA）・フッター
