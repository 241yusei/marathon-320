# マラソン3:20切りプロジェクト — Webダッシュボード

藤井勇成のマラソン挑戦（サブスリー最終目標／当面 3:19:59）を、Apple（iPhone 製品ページ）の
設計言語で 1 枚に集約したヘルスケア・ダッシュボードサイト。**今週の目標・計画／ランニング結果・
フィードバック／目標達成に必要な情報**を蓄積し、データが来るたびに更新していく。

## 開き方

ビルド不要の静的サイト。`index.html` をブラウザで開くだけ。
ローカルサーバーで見る場合:

```bash
python3 -m http.server 8931
# → http://localhost:8931
```

## 構成

```
index.html        ← セクション骨格（中身は JS が描画）
css/style.css     ← Apple 風スタイル（ダーク/ライト交互・大見出し・スクロール連動）
js/data.js        ← ★単一データソース（ここだけ編集すれば全体が更新される）
js/main.js        ← data.js を読み込み各セクションを描画・チャート・アニメ
sw.js             ← PWA用サービスワーカー（オフライン対応・キャッシュ管理）
```

## ★ 更新のしかた（蓄積とアップデート）

サイトの中身はすべて **`js/data.js`** から自動描画される。新しいデータが来たら
`data.js` の該当箇所を編集して保存 → ブラウザを再読み込みするだけで反映される。

| 何が来たか | data.js のどこを編集するか |
|------------|---------------------------|
| 朝のHRV/RHR/睡眠/体重 | `dailyLog` に1行追記 ＋ `metrics`・`highlights` の最新値を更新 |
| ランニング実績 | `dailyLog`（または `recentRuns`）に追記 |
| 週次の集計・7日間トレンド | `trends.series.*.data` に追記、`metrics` を更新 |
| 今週の優先事項が変わった | `week.priorities` / `week.judgment` を書き換え |
| 体重・睡眠ゲートの進捗 | `gates[].progress` を更新（減量ゲートは 0〜14日） |
| フェーズ移行 | `phases[].active` を移動、`meta.phase` を更新 |
| 最新研究 | `research` に1件追加 |
| 何か更新したら | **必ず `meta.lastUpdated` を更新**（ナビ右上・本日の指針の鮮度表示に使われる） |

> コーチング運用の詳細ルール（データ受領時にNotion・Googleカレンダーへどう連携するか等）は
> `CLAUDE.md` を参照。サブスリー攻略の科学的根拠は `docs/sub3-research.md`、
> トレーニング負荷・HRV判断の根拠は `docs/training-protocol.md`、
> 自転車クロストレーニングの根拠は `docs/bike-cross-training-research.md`、
> トレッドミルHIITの根拠は `docs/treadmill-hiit-research.md` を参照。

## セクション一覧

1. ヒーロー（目標タイム・減量・レースまでのカウントダウン）
2. なぜ、この計画なのか（燃え尽きた過去 → 履歴タイムライン）
3. 今日の指針（バイタル・関門・今日やること）
4. 今週の目標・計画（判定・優先事項・週間スケジュール・次回メニューの根拠）
5. ランニング結果・フィードバック（日次ログ・直近の走り・振り返り）
6. 7日間トレンド（HRV / 安静時HR / 睡眠 / 距離の切り替えチャート）
7. ゲート / 開放条件（減量・Phase移行）
8. ハイライト（最重要6指標のカード）
9. 現在地 / Status（全指標グリッド・詳細は折りたたみ）
10. 計画ロードマップ（Phase 0〜3＋テーパー）
11. 3:20切りに必要なもの（必要指標・タイム短縮の積み上げ）
12. 栄養・心拍ゾーン（詳細は折りたたみ）
13. 最新研究の反映（詳細は折りたたみ）
14. 強みとリスク
15. 次の目標（CTA）・技術仕様（折りたたみ）・フッター
